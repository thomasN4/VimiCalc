Rule 110 on VimiCalc
====================

Elementary cellular automaton. Each generation is one spreadsheet row.

Neighbors of the previous generation (relative cell refs from the cell below):
  kh = up-left  (L)
  k  = up       (C)
  kl = up-right (R)

Next state — blank-safe RPN (empty neighbors must be coerced to 0, because
VimiCalc treats blank as the multiplicative identity 1 for `*`):

  k 0 + kl 0 + + k 0 + kl 0 + * + kh 0 + k 0 + * kl 0 + * + 2 mod

  ≡ (C + R + C*R + L*C*R) mod 2

This matches Rule 110's truth table (outputs 0,1,1,0,1,1,1,0 for LCR=000..111).

Seed: a single 1 on the right edge of row 1; everything else 0.

Regenerate the full demo (on a checkout with tests):

  ./gradlew test --tests 'vimicalc.model.Rule110DemoTest'

Open in the app:

  ./gradlew run --args="demos/rule110.json"

The original C24 formula in this file did NOT match Rule 110's truth table.
It is omitted; use the RPN above instead. The conditional/macro technique
(`<cond{then}{else}`, `$D24@b`, etc.) still works for other steppers.
