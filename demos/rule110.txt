Rule 110 on VimiCalc
====================

Elementary cellular automaton. Each generation is one spreadsheet row.

Neighbors of the previous generation (relative cell refs from the cell below):
  kh = up-left  (L)
  k  = up       (C)
  kl = up-right (R)

Next state — blank-safe RPN (empty neighbors must be coerced to 0, because
VimiCalc treats blank as the multiplicative identity 1 for `*`):

  k 0 + kl 0 + + k 0 + kl 0 + * + kh 0 + k 0 + * kl 0 + * + 2 mod

  ≡ (C + R + C*R + L*C*R) mod 2

This matches Rule 110's truth table (outputs 0,1,1,0,1,1,1,0 for LCR=000..111).

Seed: a single 1 on the right edge of row 1; everything else blank/0.

Verify / regenerate goldens
---------------------------

  ./gradlew test --tests 'vimicalc.model.Rule110DemoTest'

  # rewrite demos/rule110.json and demos/rule110-formulas.json intentionally:
  ./gradlew test --tests 'vimicalc.model.Rule110DemoTest' -DupdateRule110Demos=true

Open in the app:

  ./gradlew run --args="demos/rule110.json"
  ./gradlew run --args="demos/rule110-formulas.json"

Re-record a headless implement+run session (needs Xvfb, e.g. on kubuntu):

  ./gradlew test --tests 'vimicalc.controller.Rule110SessionDirector'

--- Live build recipe (in the app) ---

From a blank sheet (cursor starts at B2):

  1. Narrow columns (optional): :resCol -72, move right, repeat.
  2. Seed: put a single 1 on the right edge of row 1 (e.g. P1). Blanks elsewhere
     are fine — the formula coerces empty neighbors to 0.
  3. On A2 press = and enter the RPN above, then Enter.
  4. yy to yank the formula cell; l pp across the rest of the row
     (relative kh/k/kl re-bind per cell).
  5. On the next rows, paste the same formula across each generation.

Macro stepper (with paced playback)
-----------------------------------

After the seed and one formula row exist, and the formula is still on the
clipboard (yy once):

  qa          " start recording macro a
  16h j       " from past the end of a filled row: back to col A, next gen
  16@f        " if macro f is: pp l   (paste + move right), width=16
  q           " stop recording

  ;macroDelay 120
  10@a        " grow 10 generations, one keystroke per 120ms
  ESC         " aborts a paced replay if needed

Older notes used a C24 formula that did NOT match Rule 110's truth table; use
the RPN above instead. Conditionals (`<cond{then}{else}`, `$D24@b`, …) still
work for more elaborate steppers.
